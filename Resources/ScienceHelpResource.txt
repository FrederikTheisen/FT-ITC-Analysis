## Baseline and Integration
The integrated heat for each injection is calculated from the baseline-corrected thermogram over the selected integration window. Baseline choice and integration range are therefore coupled: changing the selected peak region can change which points are used to estimate the baseline and can also change the integrated area directly.

\appname supports three baseline strategies:
**Spline baseline:** local baseline estimates are placed around injection scopes and interpolated with a linear or smooth spline. Sparse, balanced, and dense point-density settings change the target number of control points per injection. The program adapts the actual point count to the usable baseline length, especially when integrated peak regions remove much of the available baseline.
**Polynomial baseline:** a single polynomial is fit across the thermogram while outlier points are iteratively rejected. This is a global description of baseline drift and is most appropriate when the baseline is smooth across the full experiment.
**Segmented baseline:** local polynomial segments are fit in the baseline regions surrounding each injection and blended across integration windows. This is useful when baseline drift changes locally and should not be forced into one global polynomial.

When the setting to discard integration regions for baseline fitting is enabled, points inside selected peak windows are excluded from the baseline fit. This reduces peak contamination in the baseline estimate, but it also means integration edits can affect subsequent baseline calculations. Locking the processor or spline points can be useful when the desired baseline should be preserved.

## Peak Error Bars
The error bar drawn on each injection point represents the program's estimate of the standard uncertainty of that injection's integrated heat, \sigma{q,i}. The value is computed as the combination of an integration-noise contribution and a baseline-level contribution:

                                \sigma{q,i} = √( \sigma{q,i,int}^2^ + \sigma{q,i,bl}^2^ )

Integration noise contribution:

                                \sigma{q,i,int} = \sigma{p,i} · ∆*t* · √*N*{int,eff}

Here, \sigma{p,i} is the baseline RMS noise level estimated from non-integrated points surrounding injection *i*, ∆*t* is the sampling interval, and *N*{int,eff} is an effective number of integrated samples. *N*{int,eff} is based on the actual number of integrated samples (*N*{int}), but adjusted to compensate for autocorrelation in the baseline noise because neighboring samples are not fully independent.

Baseline level contribution:

                                \sigma{q,i,bl} = \sigma{p,i} · (*T*{int} / √*N*{bl,eff})

Here, *T*{int} is the integration length in seconds, and *N*{bl,eff} is an effective number of baseline samples used to estimate the baseline level. As above, *N*{bl,eff} is adjusted to compensate for autocorrelation in the baseline noise.

## Parameter Uncertainties
The uncertainty reported for each fitted model parameter is a statistical uncertainty estimated by repeatedly refitting the dataset after adding realistic measurement noise. \appname provides two resampling-based approaches.

**Residual bootstrapping:** Fit residuals from the optimal solution are treated as an empirical description of measurement noise. Synthetic data sets are generated by resampling or adding this noise and refitting to obtain a distribution for each parameter.
**Leave-One-Out Refitting:** The fit is repeated multiple times while removing one data point at a time. Data points are injections for single-experiment fits, or entire data sets for global analyses.

For each parameter such as *N*, \dh, or \kd, \appname collects the distribution of fitted values across those refits. The reported error for a parameter is derived from that distribution, typically as a standard deviation.
The parameter values reported are not averages. They are the single best-fit values that minimize the loss function. Only the uncertainties are derived from the resampling-and-refitting scheme.
# Derived quantities
Derived parameters such as \dg and \tds are also reported with errors. These are computed from the refitted parameter distributions in the same way as the fitted parameters, by evaluating the derived quantity for each refit and taking the spread of that distribution.
# Concentration Uncertainty Propagation
If concentration uncertainties are provided for cell and syringe solutions, \appname can propagate these into parameter uncertainties by allowing refits to include concentration perturbations within the entered uncertainty bounds.

## \multexp and Analysis Results
When you analyze multiple experiments at the same time using \multexp, a combined Analysis Result is added to the data list.
Parameters in this result may be shared across experiments, fitted per experiment, or made dependent on an experimental variable such as temperature under the selected constraint scheme.
The reported global parameters are not averages. They are the single best-fit values or dependence descriptors that minimize the total mismatch across the included datasets.

Analysis Results keep the solver metadata, model options, parameter constraints, individual replicate rows, and evaluated thermodynamic quantities together. When temperature dependence is present, evaluated values are calculated at the selected evaluation temperature; otherwise they are evaluated at the mean experiment temperature. The result exporter can write either one summary row per result or all replicate rows. Summary rows aggregate parameter values across the result's replicate solutions, while all-row exports preserve the per-experiment fitted values and losses.

Advanced result tabs are shown only when the result contains the required information. Temperature-dependent structuring analysis requires a sufficient temperature span. Ionic-strength analysis requires salt metadata and a meaningful ionic-strength variation. Protonation analysis requires buffer metadata that varies across the included experiments.

## Tandem Experiments
Tandem experiments represent a titration continued after one or more syringe reloads. In a merged tandem dataset, the injections from all selected segments are treated as one continuous titration, but each segment can have its own starting active cell and titrant concentrations.

The standard concatenation path joins the thermograms, shifts injection times, preserves injection bookkeeping, and recalculates the concentration and molar-ratio progression across the combined titration. This is appropriate when the reload can be treated as a continuation without explicit back-mixing correction.

Back-mixing mode uses an active-cell plus dead-volume model between segments. During each segment, injected titrant displaces mixed solution into the dead or overflow volume. Between segments, the model can remove displaced titrated solution and then mix a selected fraction of the remaining dead volume back into the active cell. This changes the starting active cell and titrant concentrations for the next segment. The merged dataset stores these segment concentrations so fitting uses the tandem mass balance rather than treating every reload as a fresh experiment.

## Model Formalism
The analysis models in \appname are implemented in accordance with the MicroCal/Origin formal definitions. The implementation is also extended to concentration bookkeeping and dilution correction during fitting. Parameter values from \appname are therefore intended to be directly comparable to MicroCal-style analyses, although differences in baseline handling can still introduce small shifts in fitted values.

# Syringe Correction Mode
In the default MicroCal-style interpretation, the fitted N-term can absorb apparent error in the effective concentration or activity of the species in the cell. This means that N is not always a pure stoichiometric parameter unless concentrations and active fractions are known accurately.
Syringe correction mode provides an alternative parameterization in which the fitted correction factor is applied to the titrant concentration delivered from the syringe instead of being interpreted primarily on the cell side. The thermogram integration and heat values are unchanged; the difference is entirely in the mass-balance interpretation used during fitting.
When this mode is enabled, the fitted parameter is reported as α{syringe}. Results obtained with and without syringe correction mode should therefore not be interpreted as strictly equivalent parameterizations of the same fit.
