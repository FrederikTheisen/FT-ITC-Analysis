## About \appname
\appname is a macOS desktop application for processing, analyzing, and presenting isothermal titration calorimetry (ITC) data. It is a standalone graphical application that can be installed and launched like any other macOS program. All core workflows are available through the interface; no programming or scripting is required.
# What you can do with it
**Work on your own computer:** process and analyze ITC data away from the instrument workstation, whether in the office, at home, or while travelling.
**Import data from multiple sources:** raw data from MicroCal-style instruments, exports from TA Instruments, PEAQ-ITC project files, integrated heats, and \appname project files.
**Process thermograms:** define baselines and integration regions using automated tools, with manual adjustment where needed.
**Fit models:** perform single-experiment fitting or global analysis across multiple experiments, including optional temperature-dependent constraints where applicable.
**Export figures:** generate publication-ready plots in the classical calorimetric style, with optional residual plots for diagnostics.

## Getting Started
# Loading Data
When the program opens, press **\enter** to browse for files or press **\space** to load the most recently used project or data file(s). After loading, inspect the dataset in the **Overview** tab.
The overview displays experiment metadata such as instrument information, experiment date and duration, and cell and syringe concentrations. Concentrations, temperature, and other experiment attributes can be edited in the experiment details view.
# Compatible File Formats
\appname can read the following file types:
**MicroCal-style raw data:** `.itc`
**TA Instruments NanoAnalyze exports:** `.TA`
**PEAQ-ITC project files:** `.apj`
**Integrated heats:** `.dat`, `.aff`, `.dh`
**\appname project files:** `.ftitc`
Drag compatible files into the app window, or use **File > Open…** to select them manually.
Once loaded, datasets behave the same in the application independent of instrument type. \appname does not modify raw data files. To make a project portable across computers, save it as a `.ftitc` file.

## Processing Data
Processing is the first step in ITC analysis. It consists of defining the baseline and the integration region for each injection.
Integrated heats are calculated by numerically integrating the baseline-corrected thermogram over the selected integration window. In the **Processing** view, you can zoom using the control in the lower-left corner or by dragging a zoom region. To focus on a specific injection, double-click within its injection scope.
# Selecting a Baseline Type
\appname supports two baseline interpolators.
**Spline baseline:** interpolates between baseline handle points. Handle points are placed automatically using averages of baseline regions around injections. The spline can be either smooth or linear. In handle mode, tangents can be adjusted, additional handle points can be added, and existing points can be moved vertically to improve the baseline.
**Polynomial baseline:** fits a polynomial iteratively while rejecting outlier points such as peaks. This method offers less manual control but can perform well on low-noise thermograms. Lower degrees give a less flexible baseline and higher degrees give more flexibility but increase the risk of overfitting.
By default, both baseline methods exclude points inside the integration regions. As a result, changing integration start or end positions also changes the baseline fit.
# Setting the Integration Region
Choosing an appropriate integration region is important both for accuracy and for noise reduction. Integrating over the full injection scope often adds unnecessary noise.
\appname allows you to adjust all integration regions together using the **integration start delay** and **region length** controls. Time-based and factor-based integration lengths are available, and **\peakfit** can be used when you want the program to estimate integration end points from fitted peak shapes.
You can also edit injections individually by dragging the blue region markers. When zoomed to a single injection, pressing **\space** copies the current integration length to the next injection and advances the view.

## Analyzing Data
Model fitting is performed in the **Analysis** tab. \appname currently supports four standard models:
**One-set-of-sites:** single binding process with one class of sites.
**Two-sets-of-sites:** two independent classes of binding sites fitted simultaneously.
**Competitive binding:** binding model including a competing species.
**Dissociation:** model describing dissociation of a preformed complex during titration.
All models are implemented according to MicroCal-style formalism.
# Single-Experiment Analysis
The default workflow is to analyze one experiment at a time. Select a model, set fitting options, and provide reasonable initial parameter guesses. Injections can be excluded prior to fitting if needed.
# Syringe Correction Mode
In most ITC workflows, a fitted N-value is interpreted in the traditional MicroCal-style way and mainly compensates for uncertainty in the effective concentration of the species placed in the cell.
Syringe correction mode is intended for cases where the main uncertainty is instead in the concentration of the material loaded in the syringe. When this mode is enabled, the fitted stoichiometry term is interpreted as a correction of the syringe-side titrant concentration during fitting rather than as a correction of the cell-side concentration.
# \multexp
You can analyze multiple experiments at the same time using **\multexp** mode. In this mode, all included datasets are analyzed together and a combined Analysis Result is added to the data list.
If the experiments were performed at different temperatures, the analysis result can consider this temperature variation to estimate temperature-dependent behavior, including the heat capacity change upon binding where applicable. Analysis results can also enable further analyses depending on temperature and buffer or salt conditions.
# Global Fitting
\appname can fit multiple experiments simultaneously using global optimization. Global fitting is useful for difficult interactions and for enforcing constraints across experiments, for example shared parameters across replicates, concentrations, or conditions.
A typical workflow is to load and process the relevant experiments, mark the experiments you want to include, enable global fitting in the **Analysis** tab, choose which parameters are shared and which are allowed to vary, and then run the fit.

## Presenting Data
\appname is designed to produce consistent, publication-oriented plots directly from raw data.
# Final Figure
The publication figure layout follows the classical ITC presentation style: a thermogram together with integrated heats, model overlay, and optional residuals.
Figures are generated from the current processing state and analysis result. Changing baseline or integration settings updates integrated heats, which in turn updates the fit and the final figure.
# Exporting Data
\appname includes export functions for processed results. Depending on the export tool, you can export integrated heats, associated uncertainties, raw power-vs-time traces, and fitted parameters from multi-experiment and global analyses.
Export of data is done under **File > Export** in the menu bar. Fitted parameters can also be copied from the analysis result view.

## Additional Features
# \designer
The \designer supports planning ITC experiments from within \appname. It is intended for quickly checking experimental feasibility and setting up reasonable titration conditions before collecting data.
# Experiment Attributes
Experiment attributes are metadata attached to each dataset. They are used for derived analyses and for applying constraints in multi/global fitting. Attributes can be edited in the experiment details view and are stored in `.ftitc` project files.
# Buffer Subtraction
Buffer subtraction is provided as a dedicated tool. Select a reference buffer experiment, select one or more target experiments, and apply subtraction to generate buffer-corrected datasets. Buffer subtraction operates on loaded data within the project and does not modify raw data files.
# Tandem Experiment Concatenation
Tandem experiments can be collected by refilling the syringe during an ongoing sequence, producing multiple segments that should be treated as a single continuous titration. The tandem experiment merger combines these datasets into a single merged experiment.
The merger tool preserves injection bookkeeping and concentration tracking across segments, so the merged dataset behaves like a single experiment in processing and analysis. A back-mixing mode is also available for cases where partial mixing during reload should be taken into account.
